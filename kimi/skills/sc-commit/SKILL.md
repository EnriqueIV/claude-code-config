# Ticket-Aware Git Operations

## Purpose

Handle commits, branch creation, and PRs with automatic ticket code integration. Always show a preview and wait for explicit user approval before executing ANY git mutation.

## Supported Ticket Systems

- **ClickUp**: codes like `868guc790` → auto-generates `https://app.clickup.com/t/{code}`
- **Jira/Bitbucket**: codes like `PROJ-123` → asks for workspace base URL if unknown

## Ticket Detection Priority

1. Current branch name (extracts ticket code automatically)
2. Provided by user in their message
3. Ask the user if neither is available

## Output Formats

### Branch names
```
{ticket}-{verb}-{description}
```
Example: `868guc790-implement-export-csv`

### Commit titles
```
{ticket}: {imperative description}
```
Example: `868guc790: implement export CSV endpoint`

### PR descriptions
Single plain-text paragraph + `Ticket: {url}` — no sections, no bullets, no emojis, no markdown.

Example:
```
Adds the ability for videos to begin playback at a specific timestamp,
allowing content editors to configure where viewers start watching.

Ticket: https://app.clickup.com/t/868guc790
```

## Behavioral Flow

1. **Detect**: Find ticket from current branch name or ask user.
2. **Inspect**: Show `git diff --stat` and changed files.
3. **Propose**: Show branch name / commit message / PR description for approval.
4. **Execute**: ONLY after user approves each step with explicit confirmation ("yes", "ok", "dale", "go").

## What This Skill NEVER Does

- Will **NOT** auto-commit without showing the message first.
- Will **NOT** push or create PRs without your approval.
- Will **NOT** add AI references, "Generated by Kimi", or Co-Authored-By to anything.
- Will **NOT** use markdown, bullets, sections, or emojis in PR descriptions.
- Will **NOT** run `git commit` directly — always propose and wait.

## Examples

### Commit only
```
User: commit these changes
→ Detects ticket from current branch 868guc790-implement-...
→ Shows: git diff --stat
→ Proposes: "868guc790: implement video start time playback"
→ Waits for your OK → commits
```

### Branch + commit + PR
```
User: create a branch and PR for this
→ Asks what you're building
→ Proposes: "868guc790-add-export-button"
→ Waits for your OK → creates branch
→ Commits current changes with ticket prefix
→ Composes PR title and plain description
→ Appends: "Ticket: https://app.clickup.com/t/868guc790"
→ Waits for your OK → creates PR via gh
```

## Bare Commands Rule

When running git commands, use bare commands in the current working directory. NEVER use absolute paths:

```bash
# ✅ Correct
git diff --stat
git status
git checkout -b 868guc790-fix-auth

# ❌ Wrong
git -C /Users/roger/git/my-project diff --stat
cd /Users/roger/git/my-project && git status
```
